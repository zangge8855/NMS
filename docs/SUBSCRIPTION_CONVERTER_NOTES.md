# 订阅输出与客户端导入说明

> 更新时间：2026-03-09  
> 文件名保留为 `SUBSCRIPTION_CONVERTER_NOTES.md` 仅为兼容旧引用；当前版本已不再依赖外部订阅转换器。

## 中文

### 当前结论

- NMS 直接生成并提供订阅输出
- `Clash` / `Mihomo` 使用 NMS 内置生成的 YAML 配置地址
- `v2rayN / Raw / Native / Reconstructed` 继续由 NMS 直接生成或透传
- `sing-box` 当前保留客户端导入链接方式，不依赖外部转换器

### 现在有哪些订阅类型

- `v2rayN / v2rayNG`
- `Raw`
- `Native`
- `Reconstructed`
- `Clash YAML`
- `Mihomo YAML`
- `sing-box`

### 系统设置里还需要配置什么

只需要配置：

- `订阅公网地址（SUB_PUBLIC_BASE_URL / publicBaseUrl）`

作用：

- 让管理端生成的公开订阅地址稳定使用正式域名或公网地址
- 避免页面里出现 `localhost`、内网 IP 或测试地址

当前版本不再需要：

- `转换器地址`
- `Clash 模板 URL`
- `sing-box 模板 URL`

### 管理端与客户端的展示边界

- 管理端仍然显示完整订阅链接，方便分发给用户
- 订阅内容里的节点显示名会做脱敏
- 节点标签允许保留入站备注、协议简称、端口或去重编号
- 节点标签不会暴露站点名、网站域名、邮箱等敏感展示信息

注意：

- 协议连接本身必须使用的真实目标地址仍会保留在协议字段中，否则客户端无法连接
- 被隐藏的是“显示层敏感信息”，不是协议本身必需参数

### Clash / Mihomo 规则结构

当前内置输出使用 MetaCubeX `meta-rules-dat` 的 `mrs` 规则提供器，不再使用 `txt` 规则源。

当前精简策略组：

- `PROXY`
- `AUTO`
- `GLOBAL`
- `TELEGRAM`
- `DOMESTIC`

内置目标仍包含：

- `DIRECT`

当前主要规则提供器：

- `private-domain`
- `icloud-domain`
- `apple-domain`
- `google-domain`
- `proxy-domain`
- `telegram-domain`
- `cn-domain`
- `private-ip`
- `telegram-ip`
- `cn-ip`

当前主规则顺序：

- `RULE-SET,private-domain,DIRECT`
- `RULE-SET,icloud-domain,DIRECT`
- `RULE-SET,apple-domain,DIRECT`
- `RULE-SET,google-domain,GLOBAL`
- `RULE-SET,telegram-domain,TELEGRAM`
- `RULE-SET,proxy-domain,GLOBAL`
- `RULE-SET,cn-domain,DOMESTIC`
- `RULE-SET,private-ip,DIRECT`
- `RULE-SET,telegram-ip,TELEGRAM`
- `RULE-SET,cn-ip,DOMESTIC`
- `MATCH,GLOBAL`

当前默认策略含义：

- 国内和私网优先直连或走国内策略
- Telegram 独立分组
- 其他国际流量默认走 `GLOBAL`

### 当前支持范围

可进入 Clash / Mihomo YAML 的协议：

- `vmess`
- `vless`
- `trojan`
- `shadowsocks`

当前已处理的常见传输：

- `tcp`
- `ws`
- `grpc`
- `http/h2`
- `tls`
- `reality`

当前不会进入 Clash / Mihomo YAML 的内容：

- `hy2`
- `tuic`
- `httpupgrade`
- `xhttp`
- `kcp`
- `quic`

这类节点不会影响普通订阅，只是不会写进当前版本的 Clash / Mihomo YAML。

### 客户端导入建议

- `v2rayN / v2rayNG`：优先使用普通订阅地址
- `Clash Verge / Mihomo Party`：使用 `Clash YAML` 或 `Mihomo YAML`
- `sing-box`：使用页面提供的导入链接或普通订阅地址按客户端方式导入

### 相关页面

- `订阅中心`
- `用户中心 / 开通订阅结果弹窗`
- `账号管理 / 开通订阅结果弹窗`

# Subscription Outputs and Client Import Notes

> Updated: 2026-03-09  
> The filename `SUBSCRIPTION_CONVERTER_NOTES.md` is kept only for backward compatibility. Current versions no longer depend on an external subscription converter.

## English

### Current State

- NMS generates and serves subscription outputs directly
- `Clash` and `Mihomo` use built-in YAML subscription URLs generated by NMS
- `v2rayN / Raw / Native / Reconstructed` are still generated or proxied directly by NMS
- `sing-box` keeps the import-link workflow and does not depend on an external converter

### Available Subscription Types

- `v2rayN / v2rayNG`
- `Raw`
- `Native`
- `Reconstructed`
- `Clash YAML`
- `Mihomo YAML`
- `sing-box`

### What Still Needs to Be Configured

Only this is still recommended:

- `Public subscription base URL (SUB_PUBLIC_BASE_URL / publicBaseUrl)`

Purpose:

- keep generated public subscription links stable on your formal domain or public address
- avoid exposing `localhost`, private IPs, or temporary test addresses

No longer needed in current versions:

- `Converter base URL`
- `Clash template URL`
- `sing-box template URL`

### Display Boundary Between Admin UI and Client UI

- The admin interface still shows full subscription URLs so they can be distributed to users
- Node display names inside subscription content are privacy-masked
- Node labels may keep inbound remarks, protocol abbreviations, ports, or deduplicated suffixes
- Node labels must not expose site name, domain, or email as display text

Important:

- Real target addresses required by a protocol must still stay in protocol fields, otherwise clients cannot connect
- What is hidden is the sensitive display layer, not required protocol parameters

### Clash / Mihomo Rule Structure

The built-in output now uses MetaCubeX `meta-rules-dat` `mrs` rule providers instead of `txt` rule sources.

Current compact policy groups:

- `PROXY`
- `AUTO`
- `GLOBAL`
- `TELEGRAM`
- `DOMESTIC`

Built-in routing targets still include:

- `DIRECT`

Current primary rule providers:

- `private-domain`
- `icloud-domain`
- `apple-domain`
- `google-domain`
- `proxy-domain`
- `telegram-domain`
- `cn-domain`
- `private-ip`
- `telegram-ip`
- `cn-ip`

Current primary rule order:

- `RULE-SET,private-domain,DIRECT`
- `RULE-SET,icloud-domain,DIRECT`
- `RULE-SET,apple-domain,DIRECT`
- `RULE-SET,google-domain,GLOBAL`
- `RULE-SET,telegram-domain,TELEGRAM`
- `RULE-SET,proxy-domain,GLOBAL`
- `RULE-SET,cn-domain,DOMESTIC`
- `RULE-SET,private-ip,DIRECT`
- `RULE-SET,telegram-ip,TELEGRAM`
- `RULE-SET,cn-ip,DOMESTIC`
- `MATCH,GLOBAL`

Meaning of the default policy:

- domestic and private traffic prefer direct or domestic handling
- Telegram is isolated into its own group
- most other international traffic falls back to `GLOBAL`

### Current Support Range

Protocols that can enter Clash / Mihomo YAML:

- `vmess`
- `vless`
- `trojan`
- `shadowsocks`

Currently handled common transports:

- `tcp`
- `ws`
- `grpc`
- `http/h2`
- `tls`
- `reality`

Currently excluded from Clash / Mihomo YAML:

- `hy2`
- `tuic`
- `httpupgrade`
- `xhttp`
- `kcp`
- `quic`

These do not affect normal subscriptions. They are simply skipped from the current Clash / Mihomo YAML output.

### Recommended Client Import Path

- `v2rayN / v2rayNG`: use the normal subscription URL first
- `Clash Verge / Mihomo Party`: use `Clash YAML` or `Mihomo YAML`
- `sing-box`: use the provided import link or import the normal subscription URL according to the client workflow
